# BUG-001 — indexing-service crashes on startup: Weaviate connection at import time

**Priority:** P1  
**Status:** open  
**Phase:** 2 (post-completion defect)

---

## Context

When `docker compose up` is run, `indexing-service` exits with code 1 and enters a restart loop. The traceback ends with:

```
weaviate.exceptions.WeaviateConnectionError: Connection to Weaviate failed.
Details: Error: [Errno 111] Connection refused.
```

There are two root causes that compound each other.

---

## Root Cause 1 — Module-level Weaviate connection

`services/indexing-service/src/main.py`, line 25:

```python
_writer = WeaviateWriter(weaviate_url=settings.weaviate_url)
```

`WeaviateWriter.__init__` calls `weaviate.connect_to_custom()` synchronously, which immediately opens a TCP connection to Weaviate. Because this is module-level code, it executes the moment uvicorn imports `src.main` — before the `lifespan` function runs and before any readiness check can happen.

**Fix:** Move instantiation into the `lifespan` startup block so the connection is made after the process is fully initialised and after Docker's health-based ordering has taken effect.

---

## Root Cause 2 — docker-compose depends_on uses presence, not health

`infrastructure/docker/docker-compose.yml`, `indexing-service` block:

```yaml
depends_on:
  - weaviate
```

This waits only for the Weaviate *container* to exist, not for Weaviate to pass its healthcheck (`/v1/.well-known/ready`). Weaviate takes ~2 s to elect a Raft leader and open its REST API after the container starts. The indexing service starts during that window.

**Fix:** Switch to condition-based dependency:

```yaml
depends_on:
  postgres:
    condition: service_healthy
  weaviate:
    condition: service_healthy
  elasticmq:
    condition: service_started   # no healthcheck defined
```

---

## Files Affected

| File | Change |
|------|--------|
| `services/indexing-service/src/main.py` | Move `_writer` init from module scope into `lifespan` |
| `infrastructure/docker/docker-compose.yml` | Add `condition: service_healthy` for weaviate dependency |
